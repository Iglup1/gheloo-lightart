# Habbo Light Ball Color System - Complete Light-Only Color Schema
# Version: v4 - no black/dark-multiply model
# Purpose: GitHub / Codex integration
# Language: practical technical guide

This document describes a clean color schema for Habbo-style light ball sprites.

Important correction:
Black cannot be created by stacking these lights. In light mixing, black means no light. Since the Habbo lamps are glowing transparent sprites, stacking them creates more visible light, not black.

So the system should focus on:

    bright colors
    pastel colors
    warm/cold colors
    aura + core effects
    side-overlaps
    softer/deeper colors by reducing brightness
    muted colors by mixing opposite colors carefully

Do not tell the script that true black can be made from these lights.

---

# 1. Base idea

Every light ball is a semi-transparent radial sprite.

A layer has:

    color
    size
    count
    position / offset
    opacity
    z-order

The final visible light is made by stacking layers.

Basic rule:

    more layers = brighter
    more layers of one color = more of that color
    white layers = lighter / pastel / hotter center
    XL/XXL = big aura
    S/M = small bright core / hotspot
    side-offset = directional glow / gradient

# 3. Sprite sizes

Approximate analyzed values:

| Size | Sprite bbox | Approx radius | Center alpha | Main use |
|---|---:|---:|---:|---|
| S | 63x64 px | 31-32 px | 128 | tiny hotspot / center color |
| M | 127x128 px | 63-63.5 px | 128 | core glow |
| L | 191x192 px | 95-95.5 px | 128 | middle glow |
| XL | 250/251x251 px | about 125 px | 127-128 | large aura |
| XXL | 252/253x253 px | about 126 px | 127-128 | large aura / side aura |

Center alpha:

    128 / 255 = about 50.2% visible

Stacking the same centered sprite:

| Layers | Center visible strength |
|---:|---:|
| 1 | about 50.2% |
| 2 | about 75.2% |
| 3 | about 87.6% |
| 4 | about 93.8% |
| 5 | about 96.9% |
| 6+ | almost saturated |

Formula:

    combinedAlpha = 1 - (1 - alpha)^layerCount

with:

    alpha = 128 / 255

# 4. What is possible and impossible

## Possible

You can make:

    many bright colors
    many pastel colors
    warm colors
    cold colors
    magic purple/pink colors
    lime/toxic colors
    amber/gold colors
    soft/deep-looking auras
    layered gradients
    side glows
    brighter centers
    white-hot centers

## Not possible with light stacking only

You cannot create true black by stacking lights.

Why:

    black = no light
    these sprites add visible light
    stacking increases visible light

## How to make something look darker without black

Use:

    fewer layers
    lower opacity
    bigger sprite without small hotspot
    blue/purple aura for visually deeper color
    less white
    no center hotspot
    spread color over XL/XXL only

This gives a dim or deep-looking glow, but not true black.

# 5. Core color rules

## Rule 1 - More of a color means more of that color

Example:

    1x orange + 1x white = pastel orange
    2x orange + 1x white = more orange
    3x orange + 1x white = strong orange
    1x orange + 2x white = lighter pastel orange

## Rule 2 - White makes colors lighter

White is the main tool for:

    pastel color
    hot center
    candle center
    ice center
    soft glow

## Rule 3 - Yellow/orange/red make colors warmer

Warm direction:

    yellow -> orange -> red

Use for:

    candle
    fire
    gold
    sunset
    cozy lamp

## Rule 4 - Cyan/blue make colors colder

Cold direction:

    cyan -> blue

Use for:

    ice
    neon blue
    water
    electric light

## Rule 5 - Purple creates magic colors

Purple with white:

    violet / soft purple

Purple with orange/white:

    peach magic core
    pink-orange center inside purple aura

Purple with blue:

    deeper violet/blue aura

## Rule 6 - Opposite colors mute each other

Mixing opposite-ish colors can make the glow less pure.

Examples:

    orange + blue = muted/desaturated warm-green/grayish effect
    red + cyan = pale/whitish/muted effect
    purple + yellow = muted/pinkish/grayish effect
    green + purple = muted effect

Use these carefully. They can look dirty if too many are stacked.

Use this object format:

```json
{
  "name": "Purple aura warm core",
  "tags": ["magic", "warm-core", "aura"],
  "layers": [
    { "role": "aura", "size": "XXL", "color": "purple", "count": 1, "x": 0, "y": 0, "z": 0, "opacity": 1.0 },
    { "role": "core", "size": "M", "color": "orange", "count": 2, "x": 0, "y": 0, "z": 1, "opacity": 1.0 },
    { "role": "hotspot", "size": "S", "color": "white", "count": 1, "x": 0, "y": 0, "z": 2, "opacity": 1.0 }
  ]
}
```

Layer roles:

    aura = big outside color, usually XL or XXL
    field = wide main color, usually L/XL
    core = middle bright color, usually M/L
    hotspot = small center, usually S/M
    sideAura = large offset XL/XXL glow from one side

# 9. Color approximation algorithm

Use this only for quick preview swatches and naming.

It treats the recipe like a weighted color mix.

```js
const COLORS = {
  yellow: [255, 255, 0],
  cyan:   [0, 255, 255],
  white:  [255, 255, 255],
  orange: [255, 153, 0],
  purple: [204, 0, 255],
  green:  [0, 255, 0],
  blue:   [0, 51, 255],
  red:    [255, 0, 0],
};

function weightedRecipeColor(items) {
  let total = 0;
  let r = 0, g = 0, b = 0;

  for (const item of items) {
    const [cr, cg, cb] = COLORS[item.color];
    const count = item.count ?? 1;
    const roleWeight = item.roleWeight ?? 1;

    const w = count * roleWeight;
    r += cr * w;
    g += cg * w;
    b += cb * w;
    total += w;
  }

  return [
    Math.round(r / total),
    Math.round(g / total),
    Math.round(b / total)
  ];
}
```

Suggested role weights for a center-preview color:

```js
const ROLE_WEIGHTS = {
  aura: 0.35,
  field: 0.50,
  core: 1.00,
  hotspot: 1.25,
  sideAura: 0.25
};
```

Why:
The aura is large and visible around the outside, but the center color is more affected by the core and hotspot.

For a full image preview, do not use the weighted average. Use real alpha compositing.

# 10. Real render algorithm

For a real preview, render pixel-by-pixel:

    1. Create empty canvas.
    2. Sort layers by z.
    3. For each layer and count:
        - sample the sprite alpha at the pixel
        - multiply by opacity
        - composite over previous result
    4. Output final preview image.

Use normal alpha compositing.

Pseudo-code:

```js
function renderRecipe(recipe, canvasWidth, canvasHeight) {
  const sorted = [...recipe.layers].sort((a, b) => a.z - b.z);

  for (const layer of sorted) {
    for (let i = 0; i < layer.count; i++) {
      drawSpriteLayer(layer);
    }
  }
}
```

Alpha approximation:

```js
function alphaAt(size, localX, localY) {
  const s = SIZES[size];
  const cx = (s.w - 1) / 2;
  const cy = (s.h - 1) / 2;

  const dx = (localX - cx) / s.radiusX;
  const dy = (localY - cy) / s.radiusY;
  const d = Math.sqrt(dx * dx + dy * dy);

  if (d >= 1) return 0;

  return Math.round(128 * (1 - d));
}
```

Better:
Use the real alpha mask PNG/CSV for pixel-perfect output.

# 11. Full preset schema

The script should not hardcode every possible final color.
It should hardcode useful families and generate variants.

```js
const COLOR_FAMILIES = {
  warm: {
    colors: ["red", "orange", "yellow", "white"],
    aura: ["red", "orange"],
    core: ["orange", "yellow"],
    hotspot: ["yellow", "white"]
  },

  cold: {
    colors: ["blue", "cyan", "white"],
    aura: ["blue", "cyan"],
    core: ["cyan", "blue"],
    hotspot: ["white", "cyan"]
  },

  magic: {
    colors: ["purple", "blue", "red", "orange", "white"],
    aura: ["purple", "blue"],
    core: ["purple", "orange", "red"],
    hotspot: ["white", "orange"]
  },

  toxic: {
    colors: ["green", "yellow", "cyan", "white"],
    aura: ["green", "cyan"],
    core: ["green", "yellow", "cyan"],
    hotspot: ["yellow", "white"]
  },

  pastel: {
    colors: ["red", "orange", "yellow", "green", "cyan", "blue", "purple", "white"],
    aura: ["white"],
    core: ["red", "orange", "yellow", "green", "cyan", "blue", "purple"],
    hotspot: ["white"]
  }
};
```

Generate recipes:

```js
function generateFamilyRecipes(familyName) {
  const f = COLOR_FAMILIES[familyName];
  const recipes = [];

  for (const auraColor of f.aura) {
    for (const coreColor of f.core) {
      for (const hotspotColor of f.hotspot) {
        for (const auraSize of ["XL", "XXL"]) {
          for (const coreSize of ["S", "M", "L"]) {
            for (const coreCount of [1, 2, 3]) {
              recipes.push({
                name: `${familyName}-${auraColor}-${coreColor}-${hotspotColor}-${auraSize}-${coreSize}-x${coreCount}`,
                family: familyName,
                layers: [
                  { role: "aura", size: auraSize, color: auraColor, count: 1, x: 0, y: 0, z: 0, opacity: 1.0 },
                  { role: "core", size: coreSize, color: coreColor, count: coreCount, x: 0, y: 0, z: 1, opacity: 1.0 },
                  { role: "hotspot", size: "S", color: hotspotColor, count: 1, x: 0, y: 0, z: 2, opacity: 1.0 }
                ]
              });
            }
          }
        }
      }
    }
  }

  return recipes;
}
```

# 13. Soft/deep colors without black

Since black cannot be made, "dark" should be implemented as "deep/dim".

Deep/dim means:

    lower opacity
    fewer layers
    larger aura
    no white hotspot
    blue/purple/red aura colors

Examples:

| Desired look | Recipe |
|---|---|
| deep blue | XXL blue x1, opacity 0.6, no hotspot |
| deep purple | XXL purple x1 + L blue x1, opacity 0.7 |
| deep red | XXL red x1 + L red x1, opacity 0.6 |
| soft amber | XL orange x1 + M yellow x1, opacity 0.65 |
| soft green | XXL green x1, opacity 0.55 |
| dim cyan | XL cyan x1, opacity 0.5 |

Do not call these black.
They are dim colored glow.

# 15. Side-overlap system

Use side-overlap when XL/XXL should be placed partly on the side of smaller balls.

Side-overlap creates:

    directional color
    one-sided aura
    off-center gradient
    furniture placed inside each other look

Offset guide:

| Offset amount | Effect |
|---:|---|
| 0% radius | fully centered |
| 25% radius | subtle directional tint |
| 50% radius | clear side color |
| 75% radius | strong edge overlap |
| 100% radius | very soft edge touch |

Example:

```json
{
  "name": "Purple left aura with orange core",
  "layers": [
    { "role": "sideAura", "size": "XXL", "color": "purple", "count": 1, "x": -70, "y": 0, "z": 0 },
    { "role": "core", "size": "M", "color": "orange", "count": 2, "x": 0, "y": 0, "z": 1 },
    { "role": "hotspot", "size": "S", "color": "white", "count": 1, "x": 0, "y": 0, "z": 2 }
  ]
}
```

Side directions:

```js
function sideOffset(side, distance) {
  if (side === "left") return { x: -distance, y: 0 };
  if (side === "right") return { x: distance, y: 0 };
  if (side === "top") return { x: 0, y: -distance };
  if (side === "bottom") return { x: 0, y: distance };
  return { x: 0, y: 0 };
}
```

# 18. How Codex should choose the best combination

When user asks for a color:

## Step 1 - classify requested color

```js
function classifyColorName(name) {
  const n = name.toLowerCase();

  if (["orange", "amber", "gold", "fire", "warm", "candle", "sunset"].some(x => n.includes(x))) return "warm";
  if (["blue", "ice", "cold", "cyan", "aqua", "neon blue"].some(x => n.includes(x))) return "cold";
  if (["purple", "violet", "magic", "pink", "magenta"].some(x => n.includes(x))) return "magic";
  if (["green", "lime", "toxic", "mint", "nature"].some(x => n.includes(x))) return "toxic";
  if (["pastel", "soft", "light", "white"].some(x => n.includes(x))) return "pastel";

  return "custom";
}
```

## Step 2 - choose aura/core/hotspot

```js
function buildRecipeFromFamily(family, strength = "normal") {
  const strengthMap = {
    soft: { auraCount: 1, coreCount: 1, hotspotCount: 0, opacity: 0.6 },
    normal: { auraCount: 1, coreCount: 1, hotspotCount: 1, opacity: 1.0 },
    bright: { auraCount: 1, coreCount: 2, hotspotCount: 1, opacity: 1.0 },
    extreme: { auraCount: 2, coreCount: 3, hotspotCount: 2, opacity: 1.0 }
  };

  const s = strengthMap[strength] ?? strengthMap.normal;

  if (family === "warm") {
    return createCenteredRecipe("orange", "orange", "white", s);
  }

  if (family === "cold") {
    return createCenteredRecipe("blue", "cyan", "white", s);
  }

  if (family === "magic") {
    return createCenteredRecipe("purple", "orange", "white", s);
  }

  if (family === "toxic") {
    return createCenteredRecipe("green", "yellow", "white", s);
  }

  return createCenteredRecipe("white", "white", "white", s);
}
```

## Step 3 - tune the recipe

If result is not right:

    too pale -> add more core color
    too weak -> increase coreCount/hotspotCount
    too bright -> lower opacity or remove hotspot
    too warm -> add cyan/blue
    too cold -> add orange/yellow
    not enough aura -> increase aura size/count
    needs direction -> offset aura
    needs magic -> use purple aura

# 20. Very short summary

These Habbo light balls can create many colors by stacking transparent radial sprites.

The correct model:

    color recipe = layers + sizes + counts + offsets + opacity

Black is not a valid generated light color.
Black means no light.

To create more colors:

    repeat a color to make it dominant
    add white to make it lighter/pastel
    add yellow/orange/red to make it warmer
    add cyan/blue to make it colder
    add purple for magic/violet aura
    use green/yellow/cyan for toxic/nature colors
    use XL/XXL as aura
    use S/M as core and hotspot
    use side offsets for directional blends
    use lower opacity/fewer layers for dim/deep colors

Use a generator based on this schema.
